Git

Git is a version control system that allows managing source code history efficiently. It can also be downloaded to your computer.

Git runs in the terminal or the Command Prompt only. Every Git commit has their own ID, the author, the date and the Commit message. “HEAD” just points to the latest Commit.

Commands

git init

Will turn our project into a project managed by Git.

git status

Provides information about the current state.

git add .

will track changes in all files inside the Repository.

git commit -m

Adds the latest version of the code to our Branch, -m should provide a meaningful Commit message.

git branch

Lists all Branches in our Repository.

git log

Will display all Commits inside the Branch.

git checkout

Allows to select a specific Branch or a specific Commit.

git master

Takes you back to the master branch.

git reset --hard ID

ID should be the ID of the commit you want. Deletes all later commits.

git checkout -- .

Lets you jump back to the last commit.

git checkout -b

Creates a new Branch based on the latest Commit of the Branch you’re currently working in.

git checkout -b new-feature

Will create that new Branch named “new-feature”.

git merge branch-name

Merges branches.

git branch -D branch-name

Deletes branch.

Github

Github is a hosting and collaboration provider which hosts Git projects. Code is stored in repositories, which can be made public. Inside a repository are branches or “subfolders” where the code is actually stored.

A repository can have many branches. A branch contains different versions of code, commits. Each commit is a snapshot of a specific version of the code.

The “Clone or download” option in the remote Repository provides a URL that can be shared to give others users Repository access.

Commands

git remote add origin URL

remote add - Adds a remote Repository to the Git project
origin - This is the name of the remote Repository
URL - Is the URL of the remote Repository on GitHub

git push

Allows to push (i.e. upload) the content of the local Repository to the remote one.

git push -u origin master

Instructs Git to create an upstream to a specific Branch and Repository.

git remote

Displays the remote Repository in the terminal.

git branch

Displays all current Branches in the local Repository.

git branch -r

Displays the remote tracking Branch.

git fetch

Downloads information from a remote Repository to the remote tracking Branch.

git pull

Basically a combination of git fetch and git merge.
